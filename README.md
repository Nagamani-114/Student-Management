# 🎓 Student Management System

A comprehensive CRUD (Create, Read, Update, Delete) application for managing student records. Built with **Java**, **Object-Oriented Programming principles**, **Collections Framework**, and **MySQL database** persistence.

## ✨ Features

- ✅ **Add Students** - Register new students with comprehensive validation
- 📋 **View All Students** - Display complete list in formatted table
- 🔍 **Search Students** - Find students by name with keyword matching
- ✏️ **Update Students** - Modify existing student records with partial update support
- 🗑️ **Delete Students** - Remove student records by ID
- 📊 **View Statistics** - Get insights (total count, average CGPA, top performer)
- 📤 **CSV Import** - Load bulk student data from CSV files
- 💾 **Database Persistence** - All records stored in MySQL database
- 🎨 **Modern GUI** - Swing-based graphical interface with tabbed layout
- 🖥️ **Console Interface** - Text-based menu-driven interface

## 📋 Core Classes and Responsibilities

### 1. **Student.java** - Entity/Model Class
Represents the Student data model with the following attributes:

```java
private int id;           // Unique identifier (auto-generated by database)
private String name;      // Student full name (cannot be empty)
private int age;          // Student age (must be 17-30)
private String branch;    // Academic branch or department
private double CGPA;      // Cumulative GPA (0.0 - 10.0 scale)
```

**Key Methods:**
- Getters and setters for all properties
- `toString()` - Returns formatted table row with pipe separators

### 2. **StudentManager.java** - Business Logic Layer
Handles all CRUD operations and database interactions:

**CREATE Operation:**
- `addStudent(String name, int age, String branch, double cgpa)`
- Validates: Name not empty, Age 17-30, CGPA 0.0-10.0
- Throws `IllegalArgumentException` for invalid inputs

**READ Operations:**
- `getAllStudents()` - Returns `List<Student>` of all records
- `getStudentById(int id)` - Returns `Optional<Student>`
- `searchByName(String keyword)` - Returns filtered list

**UPDATE Operation:**
- `updateStudent(int id, String name, int age, String branch, double cgpa)`
- Updates all fields for a given student ID

**DELETE Operation:**
- `deleteStudent(int id)` - Removes student record, returns boolean

**Utility Methods:**
- `getTotalCount()` - Returns total number of students
- `getAverageCgpa()` - Calculates average CGPA
- `getTopStudent()` - Returns student with highest CGPA
- `loadFromCSV(String path)` - Imports students from CSV file

### 3. **DBconnection.java** - Database Connection Handler
Manages database connectivity to MySQL:

```java
Database: student_db
Host: localhost:3306
User: root
Driver: com.mysql.cj.jdbc.Driver (JDBC)
Connection Pool: Standard DriverManager (can be enhanced)
```

**Key Method:**
- `getConnection()` - Returns fresh `Connection` object
- Throws `Exception` for connection failures

### 4. **Main.java** - Console Application Entry Point
Interactive menu-driven console interface (v1.0):

**Menu Structure (8 Options):**
```
1. Add Student      - Add new student with validation
2. View All         - Display all students in formatted table
3. Search           - Search by student name (partial matching)
4. Update           - Modify student record (supports partial updates)
5. Delete           - Remove student by ID
6. View Statistics  - Show total count, average CGPA, top performer
7. Load from CSV    - Import bulk students from CSV file
8. Exit             - Quit application
```

**Features:**
- Formatted output with box-drawing characters (╔, ╚, ├, ┤, etc.)
- Input validation for all user entries
- Error handling with exception messages
- Seed data pre-loaded for demonstration
- Optional input for updates (press Enter to keep existing value)
- Real-time feedback messages

### 5. **StudentManagementGUI.java** - GUI Application Entry Point
Modern Swing-based graphical interface:

**Tabbed Interface (6 Tabs):**
- ➕ **Add Student** - Form-based student registration
- 📋 **View All** - Table display with sorting
- 🔍 **Search** - Search by name with instant results
- ✏️ **Update** - Load and modify student records
- 🗑️ **Delete** - Remove students with confirmation
- 📊 **Statistics** - Key metrics and insights

**Features:**
- Clean Swing layout with JTabbedPane
- Data tables with JTable components
- Input validation with error dialogs
- Confirmation dialogs for destructive actions
- Real-time status messages
- Professional color scheme

## 🚀 Getting Started

### Prerequisites

1. **Java Development Kit (JDK)**
   ```
   Version: Java 8 or higher
   Download: https://www.oracle.com/java/technologies/downloads/
   ```

2. **MySQL Database**
   ```
   Version: MySQL 5.7+ or MySQL 8.0+
   Download: https://dev.mysql.com/downloads/mysql/
   ```

3. **MySQL JDBC Driver**
   ```
   File: mysql-connector-java-8.0.xx.jar
   Add to classpath before compilation
   ```

4. **IDE (Recommended)**
   ```
   IntelliJ IDEA Community Edition
   Download: https://www.jetbrains.com/idea/
   ```

### Database Setup

1. **Create the database:**
   ```sql
   CREATE DATABASE student_db;
   USE student_db;
   ```

2. **Create the students table:**
   ```sql
   CREATE TABLE students (
       id INT PRIMARY KEY AUTO_INCREMENT,
       name VARCHAR(50) NOT NULL,
       age INT NOT NULL,
       branch VARCHAR(50),
       cgpa DOUBLE NOT NULL,
       created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   );
   ```

3. **Update connection credentials in DBconnection.java (if needed):**
   ```java
   private static final String URL = "jdbc:mysql://localhost:3306/student_db";
   private static final String USER = "root";
   private static final String PASSWORD = "your_password";
   ```

## 🎯 Key OOP Concepts Implemented

1. **Encapsulation**
   - Private fields with public getters/setters in Student class
   - Database credentials hidden in DBconnection class

2. **Abstraction**
   - StudentManager hides database details from console interface
   - Complex SQL operations abstracted into simple method calls

3. **Collections Framework**
   - `ArrayList<Student>` for storing student records
   - `List` interface for flexible return types
   - `Optional<Student>` for null-safe operations

4. **Exception Handling**
   - Input validation with `IllegalArgumentException`
   - Try-catch blocks for database operations
   - User-friendly error messages

5. **Java Features**
   - JDBC for database connectivity
   - Java Swing for GUI components
   - Lambda expressions (Optional.ifPresent())
   - String.format() for formatted output

## 🐛 Troubleshooting

### Issue: "Connection refused" when running application

**Solution:**
- Ensure MySQL is running
- Check database credentials in `DBconnection.java`
- Verify database `student_db` exists

### Issue: "Table 'student_db.students' doesn't exist"

**Solution:**
- Execute the SQL database setup commands above
- Ensure you're connected to `student_db` database

### Issue: "ClassNotFoundException" for MySQL driver

**Solution:**
- Add `mysql-connector-java-8.0.xx.jar` to classpath
- In IntelliJ: Project Structure → Libraries → Add MySQL JDBC JAR

### Issue: Compilation error with UTF-8 characters

**Solution:**
- Use `-encoding UTF-8` flag when compiling:
  ```bash
  javac -encoding UTF-8 studentmanagement/*.java
  ```

## 🔐 Database Security Notes

⚠️ **Current Implementation:**
- Hardcoded credentials in source code (⚠️ NOT recommended for production)
- No encryption for database passwords
- No input sanitization beyond validation

**For Production Use:**
- Use environment variables for credentials
- Implement PreparedStatements (already done for SQL injection prevention)
- Add authentication and authorization layers
- Use connection pooling (HikariCP, C3P0)
- Encrypt sensitive data

## 📋 Database Schema
```sql
CREATE TABLE students (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    age INT NOT NULL CHECK (age BETWEEN 17 AND 30),
    branch VARCHAR(50) NOT NULL,
    cgpa DECIMAL(3,2) NOT NULL CHECK (cgpa BETWEEN 0.0 AND 10.0),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 🔧 Troubleshooting

### Database Connection Issues
- Verify MySQL is running
- Check database credentials in `DBconnection.java`
- Ensure database name matches your configuration

### Compilation Errors
- Ensure Java 8 or higher is installed
- Check all source files are in `src/studentmanagement/` folder
- Verify no syntax errors in Java files

### CSV Import Errors
- Check CSV file path is correct
- Verify CSV format matches expected schema
- Ensure data types are correct (age as int, cgpa as double)
